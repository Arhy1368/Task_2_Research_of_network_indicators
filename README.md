Анализ уязвимостей
1. DNS-сервер (порт 53)
Статус: Открыт

Риски: Публичный DNS-сервер может быть использован для DDoS-атак (DNS-амплификация) или для утечки информации о сети.

Рекомендация: Ограничить доступ к порту 53 только для локальной сети, если не требуется публичное разрешение имен.

2. Веб-интерфейс (порт 80, HTTP)
Статус: Открыт

Риски:

Данные передаются в открытом виде

Логин и пароль администратора могут быть перехвачены при входе

Подверженность атакам "человек посередине" (MITM)

Рекомендация: Отключить HTTP (порт 80) в настройках роутера, использовать только HTTPS (порт 443).

3. Веб-интерфейс (порт 443, HTTPS)
Статус: Открыт (защищенный)

Риски: Безопасен при условии использования современных протоколов шифрования (TLS 1.2/1.3).

Рекомендация: Убедиться, что прошивка роутера поддерживает актуальные версии TLS.

4. Фильтрованные порты (37215, 37443)
Статус: filtered

Анализ:

Порты скрыты, что хорошо

Часто такие порты используются для удаленного управления (TR-069) или автоматических обновлений

Их наличие может указывать на служебные функции, которые лучше держать скрытыми

Рекомендация: Изучить документацию роутера для понимания назначения этих портов.
